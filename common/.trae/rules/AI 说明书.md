# AI 说明书

## 核心行为准则

AI 必须查看以下技能、工具：

### 1. 文件创建
创建文件 → [项目目录结构指引](../skills/项目目录结构指引/SKILL.md) → 确定位置 → 创建

### 2. 代码开发
全栈开发 → [全栈开发](../skills/全栈开发/SKILL.md) → 7 步流程 → [代码质量检查器](../skills/代码质量检查器/SKILL.md)

### 3. 执行命令（使用 PowerShell 语法，PowerShell 不支持 && 运算符，只能使用分号分隔）
GO 项目 → [后端开发工具集](../skills/后端开发工具集/SKILL.md)（构建/打包/运行/依赖/Swagger）
Vue 项目 → [前端开发工具集](../skills/前端开发工具集/SKILL.md)（构建/打包/运行/依赖）

### 4. 规则/技能修改
修改规则/技能 → [规则与技能创建与编辑指南](../skills/规则与技能创建与编辑指南/SKILL.md) → 检查字符数/frontmatter→提交

### 5. 质量保障
代码完成 → [代码质量检查器](../skills/代码质量检查器/SKILL.md) → 格式化 + 类型检查 + 构建验证

## 核心原则

1. **文件位置优先**：先定位置，再创建
2. **技能驱动**：优先调用对应技能
3. **规范检查**：代码完成必调质量检查器
4. **rules下的规则遵守**：≤1000 字符、中文命名、frontmatter (skills下的技能无限字符数)
5. **质量第一**：保证代码质量
6. **禁止同时操作多个文件**：一次只能操作一个文件
7. **禁止使用PowerShell的Get-Content和-replace命令修改内容**：只使用Edit工具修改文件内容
