## 核心功能概览

本技能集成了前端开发的全流程工具：

1. **组件规范检查** - 自动检查 API 路径、响应格式、弹窗规范等
2. **脚手架生成** - 自动生成符合规范的 Vue 组件和 API 服务
3. **开发命令执行** - 依赖安装、开发服务器、生产构建
4. **混合架构开发** - Element Plus + Tailwind CSS 协同开发

## 功能 1：组件规范检查与生成

### 自动检查项目

#### 1.1 API 路径检查
**触发条件**：编写前端 API 调用时

**自动检查**：
- 检测 API 路径是否使用 `//` 前缀
- 发现 `/api/user` 时提醒："⚠️ 应使用 `//user` 而非 `/api/user`"

#### 1.2 响应格式检查
**触发条件**：处理 API 响应时

**自动检查**：
- 检测是否使用 `res.code` 判断
- 检测列表字段是否使用 `items` 而非 `list`
- 检测分页参数是否使用 `page` 和 `page_size`

#### 1.3 弹窗规范检查
**触发条件**：创建新页面时

**自动检查**：
- 检测是否使用 `el-dialog` 弹窗
- 发现 `window.open()` 或路由跳转时提醒
- 提示："⚠️ 应使用单页面 + 弹窗模式，禁止打开新页面"

#### 1.4 权限标识检查
**触发条件**：编写权限控制代码时

**自动检查**：
- 检测权限标识是否使用 `update` 而非 `edit`
- 发现 `user:edit` 时提醒："⚠️ 应使用 `user:update` 而非 `user:edit`"

## 功能 2：开发命令执行

### 2.1 依赖安装

**命令**：`npm install`

**功能**：安装项目依赖包。

**使用场景**：
- 项目初始化后
- 新增 npm 包后
- 切换分支后

### 2.2 开发环境启动

**命令**：`npm run serve` 或 `npm run dev`

**功能**：启动开发服务器，支持热重载。

**使用场景**：
- 日常开发
- 功能调试
- 本地测试

### 2.3 生产环境构建

**命令**：`npm run build`

**功能**：构建生产版本，压缩和优化代码。

**使用场景**：
- 部署到生产环境
- 测试生产构建
- 发布新版本

### 2.4 类型检查

**命令**：`npm run type-check`

**功能**：执行 TypeScript 类型检查。

**使用场景**：
- 提交代码前
- 修改类型定义后
- 定期质量检查

### 2.5 代码格式化

**命令**：`npm run format` 或 `npm run lint -- --fix`

**功能**：格式化代码并修复可自动修复的问题。

**使用场景**：
- 保存文件后
- 提交代码前
- 代码审查前
