# Trae 规则与技能创建与编辑指南

## 触发时机

- ✅ 用户要求创建新的规则或技能文件时
- ✅ 用户要求修改规则或技能文件时
- ✅ 检测到新建或编辑规则/技能文件时
- ✅ 规则文件总字符数超过 1000 时
- ✅ 技能文件缺少必要字段时

## 一、命名规范

**规则文件**（`.trae/rules/`）：

- ✅ `go 代码风格规则.md`、`vue 代码风格规则.md`
- ❌ 禁止纯英文（`go_rules.md`）、禁止下划线（`go_code_style.md`）

**技能目录**（`.trae/skills/`）：

- ✅ `Gin API 业务约束与脚手架生成器/`、`Vue3 组件规范与 MCP 协同生成器/`
- ❌ 禁止下划线（`gin_api_业务/`）、禁止纯英文（`code_reviews/`）

**规则**：

1. 使用中文命名（专有名词 API、Gin、Vue、MCP 除外）
2. 使用空格分隔，禁止下划线
3. 规则文件以"规则"或"指南"结尾

## 二、规则文件规范

**位置**：`.trae/rules/<中文名称>.md`

**分类**：

### 2.1 技术规范类规则

适用于：代码风格规范、技术栈规范等

**字符限制**：⚠️ **总字符数不超过 1000 字符(超重要)**

**内容模板**：

```markdown
# [规则名称]

## 核心原则
[3-5 条核心原则]

## 具体规则
[详细说明]

```

### 2.2 行为指导类规则

适用于：AI 说明书、操作指南等

**字符限制**：⚠️ **总字符数不超过 1000 字符(超重要)**

**内容要求**：

```markdown
# [规则名称]

## 核心行为准则
[指导 AI 哪些流程和步骤该调用什么技能和规则和技能，调用顺序]

## 场景化流程
[场景 1]：流程图 + 正反示例
[场景 2]：流程图 + 正反示例
...

## 核心原则
[3-5 条核心原则]
```

**特点**：

- ✅ 使用流程图展示技能调用流程步骤
- ✅ 提供正反示例
- ✅ 面向 AI 行为指导，而非给人看

## 三、技能文件规范

**位置**：`.trae/skills/<中文名称>/SKILL.md`

**Frontmatter**：

```markdown
---
name: [技能名称]
description: [当 XX 时触发，做 XX 事情，达到 XX 目的]（50-100 字）
```

**Description 示例**：

- ✅ "当新增或修改 Go 服务时自动触发，为服务创建边界测试、运行测试并确保测试通过，提升代码质量"
- ❌ "生成测试"（太简单）

**内容模板**：

```markdown
## 功能描述
[详细说明]

## 触发条件
当检测到以下情况时自动激活：
- ✅ [触发场景 1]
- ✅ [触发场景 2]

## 核心功能
### 1. [功能 1]
[详细说明]

## 工作流程
[流程图或步骤]

```

## 五、规则 vs 技能 选择

**核心原则：技能优先** ⭐⭐⭐⭐⭐

## 六、快速检查清单

**技术规范类规则**：

- [ ] 位置：`.trae/rules/`
- [ ] 中文命名
- [ ] 总字符 ≤1000
- [ ] 有示例代码

**行为指导类规则**：

- [ ] 位置：`.trae/rules/`
- [ ] 中文命名
- [ ] 总字符 ≤1000
- [ ] 使用流程图展示技能调用步骤
- [ ] 明确核心原则
- [ ] 面向 AI 行为指导（而非给人看）

**技能文件**：

- [ ] 位置：`.trae/skills/<name>/SKILL.md`
- [ ] Frontmatter（name, description）
- [ ] Description 含触发时机、功能、效果
- [ ] 有功能说明和示例

**文档文件**：

- [ ] 位置：`<项目>/.trae/documents/`
- [ ] 中文命名
- [ ] 项目对应正确
- [ ] 有清晰的标题和结构
- [ ] 有版本历史（如适用）
