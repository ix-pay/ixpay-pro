# Trae 规则与技能创建与编辑指南

## 核心原则

- 仅使用符合 Markdown 标准的符号生成规则和技能
- 规则和技能内容的用词须严谨、准确、规范

## 一、快速检查清单（优先查看）

### 规则文件（`common/.trae/rules/`）

- [ ] **命名**：使用中文命名，词间以空格分隔，禁止使用下划线
- [ ] **字符数**：≤1000 字符（强制要求）
- [ ] **技术规范类**：须包含示例代码
- [ ] **行为指导类**：须包含流程图、正反示例，面向 AI 可读性优化

### 技能文件

- [ ] **位置**：`common/.trae/skills/<名称>/SKILL.md`
- [ ] **Frontmatter**：须包含 `name` 和 `description` 字段
- [ ] **Description**：须明确触发时机、执行动作、预期效果（50-100 字）
- [ ] **内容完整性**：须包含功能说明和示例

### 文档文件

- [ ] **位置**：`<项目>/.trae/documents/`
- [ ] **命名**：使用中文命名
- [ ] **项目归属**：确保项目目录正确
- [ ] **结构**：具备清晰的标题层级和文档结构

---

## 二、命名规范

### 规则文件（`.trae/rules/`）

- ✅ 正确示例：`go 代码风格规则.md`、`vue 代码风格规则.md`
- ❌ 错误示例：禁止纯英文命名（如 `go_rules.md`）、禁止使用下划线（如 `go_code_style.md`）

### 技能目录（`.trae/skills/`）

- ✅ 正确示例：`Gin API 业务约束与脚手架生成器/`、`Vue3 组件规范与 MCP 协同生成器/`
- ❌ 错误示例：禁止使用下划线（如 `gin_api_业务/`）、禁止纯英文命名（如 `code_reviews/`）

### 命名规则

1. **语言要求**：使用中文命名（专有名词如 API、Gin、Vue、MCP 除外）
2. **分隔符要求**：使用空格分隔词语，禁止使用下划线
3. **后缀要求**：规则文件须以"规则"或"指南"结尾

---

## 三、规则文件规范

### 技术规范类规则

**适用范围**：代码风格规范、技术栈规范等

**字符限制**：⚠️ **总字符数不得超过 1000 字符（强制要求）**

**内容模板**：
```markdown
# [规则名称]

## 核心原则
[3-5 条核心原则]

## 具体规则
[详细说明]
```

### 行为指导类规则

**适用范围**：AI 说明书、操作指南等

**字符限制**：⚠️ **总字符数不得超过 1000 字符（强制要求）**

**内容要求**：
```markdown
# [规则名称]

## 核心行为准则
[明确 AI 在各流程和步骤中应调用的技能、规则及调用顺序]

## 场景化流程
[场景 1]：流程图 + 正反示例
[场景 2]：流程图 + 正反示例

## 核心原则
[3-5 条核心原则]
```

**内容特点**：
- ✅ 使用流程图展示技能调用流程与步骤
- ✅ 提供正面与反面示例对比
- ✅ 确保 AI 易于理解和执行

## 四、技能文件规范

**位置**：`.trae/skills/<中文名称>/SKILL.md`

### Frontmatter

```markdown
---
name: [技能名称]
description: [当 XX 时触发，执行 XX 动作，达成 XX 效果]（50-100 字）
```

**Description 编写示例**：
- ✅ 正确示例："当新增或修改 Go 服务时自动触发，为服务创建边界测试、运行测试并确保测试通过，提升代码质量"
- ❌ 错误示例："生成测试"（描述过于简略）

### 内容模板

```markdown
## 核心原则
[3-5 条核心原则]

## 核心功能概览
[功能 1]、[功能 2]、[功能 3]

## 工作流程
[流程图或步骤说明]
```

## 六、Markdown 标准符号

### 标题

使用 `#` 表示标题，单个 `#` 为一级标题，支持最多六级标题

### 强调与加粗

- **斜体**：使用单个星号 `*` 或单个下划线 `_` 包裹文字
- **粗体**：使用两个星号 `**` 或两个下划线 `__` 包裹文字
- **粗斜体**：使用三个星号 `***` 或三个下划线 `___` 包裹文字

### 删除线

使用两个波浪号 `~~` 包裹文字

### 引用

使用大于号 `>` 后跟空格表示引用块

### 列表

- **无序列表**：使用 `-`、`+` 或 `*` 后跟空格表示
- **有序列表**：使用阿拉伯数字后跟点和空格表示

### 分割线

使用三个或以上的 `-`、`*` 或 `_` 表示分割线

### 链接

使用 `[链接文本](链接地址)` 格式表示超链接

### 图片

使用 `![替代文本](图片地址)` 格式表示图片

### 代码

- **行内代码**：使用单个反引号 `` ` `` 包裹代码
- **代码块**：使用三个反引号 ``` ``` 包裹多行代码

### 表格

使用 `|` 分隔列、使用 `---` 表示表头分隔线来构建表格

### 转义字符

使用反斜杠 `\` 转义特殊字符
