# IXPay Pro 部署说明

## 1. 系统架构概述

IXPay Pro 是一个基于 Go + Vue3 的前后端分离支付管理系统，采用微服务架构设计。后端使用 Gin 框架开发 RESTful API，前端使用 Vue3 + Element Plus 构建现代化界面。

### 1.1 技术栈

**后端**：
- Go 1.20+ 
- Gin Web 框架
- GORM ORM 框架
- PostgreSQL 数据库
- Redis 缓存
- JWT 认证

**前端**：
- Vue 3
- Element Plus
- TypeScript
- Vite

## 2. 环境要求

### 2.1 基础环境

| 组件 | 版本要求 | 用途 |
| --- | --- | --- |
| Go | 1.20+ | 后端开发语言 |
| Node.js | 16+ | 前端开发环境 |
| npm | 8+ | 前端依赖管理 |
| PostgreSQL | 13+ | 数据库 |
| Redis | 6+ | 缓存、会话管理 |
| Docker | 20.10+ | 容器化部署（可选） |
| Docker Compose | 1.29+ | 容器编排（可选） |

### 2.2 系统资源

- **CPU**: 至少 2 核
- **内存**: 至少 4GB
- **磁盘**: 至少 50GB 可用空间

## 3. 安装步骤

### 3.1 克隆代码仓库

```bash
git clone https://github.com/ix-pay/ixpay-pro.git
cd ixpay-pro
```

### 3.2 后端部署

#### 3.2.1 安装依赖

```bash
cd server
go mod tidy
```

#### 3.2.2 配置数据库

1. 创建 PostgreSQL 数据库：

```sql
CREATE DATABASE ixpay_pro;
CREATE USER ixpay WITH PASSWORD 'ixpay123';
GRANT ALL PRIVILEGES ON DATABASE ixpay_pro TO ixpay;
```

2. 配置 Redis：

确保 Redis 服务已启动并可访问。

#### 3.2.3 修改配置文件

编辑 `server/configs/config.yaml` 文件，配置数据库连接信息：

```yaml
# 数据库配置
database:
  type: "postgres"
  host: "localhost"
  port: 5432
  user: "ixpay"
  password: "ixpay123"
  dbname: "ixpay_pro"
  sslmode: "disable"

# Redis配置
redis:
  host: "localhost"
  port: 6379
  password: ""
  db: 0
```

#### 3.2.4 数据库迁移

```bash
cd server
go run ./cmd/ixpay-pro
```

**说明**：系统启动时会自动执行数据库迁移（根据配置文件中的 `init_seed_data` 和 `update_routes_on_start` 配置）。

#### 3.2.5 启动后端服务

```bash
# 开发环境
go run ./cmd/ixpay-pro

# 生产环境（打包后）
./build/ixpay-pro.exe
```

服务会在 `http://localhost:8586` 启动。

**说明**：
- 开发环境使用 `go run` 直接运行源代码
- 生产环境使用编译后的可执行文件（位于 `build/ixpay-pro.exe`）

### 3.3 前端部署

#### 3.3.1 安装依赖

```bash
cd ../web
npm install
```

#### 3.3.2 修改配置文件

编辑 `.env.development` 或 `.env.production` 文件，配置后端 API 地址：

```env
# 开发环境
ENV = 'development'
VITE_CLI_PORT = 8585
VITE_SERVER_PORT = 8586
VITE_BASE_API = /api/admin
VITE_BASE_PATH = http://127.0.0.1

# 生产环境
ENV = 'production'
VITE_BASE_API = /api/admin
VITE_BASE_PATH = /
```

#### 3.3.3 构建项目

```bash
# 开发环境
npm run dev

# 生产环境
npm run build
```

开发环境下，前端服务将在 `http://localhost:5173` 启动。

## 4. 配置说明

### 4.1 环境变量

| 变量名 | 描述 | 默认值 |
| --- | --- | --- |
| LOG_LEVEL | 日志级别（debug/info/warn/error） | info |
| SERVER_PORT | 服务端口 | 8080 |
| JWT_SECRET | JWT 密钥 | 随机生成 |
| JWT_EXPIRE | JWT 过期时间（秒） | 3600 |
| REDIS_HOST | Redis 主机 | localhost |
| REDIS_PORT | Redis 端口 | 6379 |
| DB_HOST | 数据库主机 | localhost |
| DB_PORT | 数据库端口 | 5432 |
| DB_USER | 数据库用户 | ixpay |
| DB_PASSWORD | 数据库密码 | ixpay123 |
| DB_NAME | 数据库名称 | ixpay_pro |

### 4.2 配置文件

主要配置文件位于 `server/configs/config.yaml`，包含以下主要部分：

- **server**: 服务器配置（端口、运行模式等）
- **database**: 数据库配置
- **redis**: Redis 配置
- **jwt**: JWT 认证配置
- **logging**: 日志配置
- **upload**: 文件上传配置

## 5. 启动和停止服务

### 5.1 后端服务

#### 5.1.1 启动服务

```bash
# 开发模式
cd server
go run ./cmd/ixpay-pro

# 生产模式
cd server
go build -o build/ixpay-pro.exe ./cmd/ixpay-pro
./build/ixpay-pro.exe
```

#### 5.1.2 停止服务

- 开发模式：按 `Ctrl + C` 停止
- 生产模式：使用 `kill` 命令停止进程

### 5.2 前端服务

#### 5.2.1 启动服务

```bash
# 开发模式
cd web
npm run dev

# 生产模式
# 使用 Nginx 等 Web 服务器部署 dist 目录
```

#### 5.2.2 停止服务

- 开发模式：按 `Ctrl + C` 停止
- 生产模式：停止 Nginx 服务

## 6. Docker 部署

**配置文件位置**：`server/docker-compose.yml`

### 6.1 构建 Docker 镜像

```bash
cd server
docker build -t ixpay-server .
```

### 6.2 使用 Docker Compose

```bash
cd server
docker-compose up -d
```

这将启动后端服务、PostgreSQL 和 Redis。

## 7. 故障排查

### 7.1 常见问题

#### 7.1.1 数据库连接失败

- 检查数据库是否已启动
- 检查数据库连接参数是否正确
- 检查数据库用户权限

#### 7.1.2 Redis 连接失败

- 检查 Redis 是否已启动
- 检查 Redis 连接参数是否正确
- 检查 Redis 密码是否正确

#### 7.1.3 JWT 认证失败

- 检查 JWT 密钥是否一致
- 检查 Token 是否过期
- 检查 Token 格式是否正确

#### 7.1.4 跨域问题

- 检查前端配置的 API 地址是否正确
- 检查后端是否已配置 CORS 中间件

### 7.2 日志查看

后端日志默认保存在 `server/logs/` 目录下，按级别分类：

- `error.log`: 错误日志
- `warn.log`: 警告日志
- `info.log`: 信息日志
- `debug.log`: 调试日志

## 8. 安全建议

1. **生产环境配置**：
   - 修改默认密码
   - 启用 HTTPS
   - 配置防火墙规则
   - 限制 API 访问 IP

2. **数据库安全**：
   - 使用强密码
   - 限制数据库访问 IP
   - 定期备份数据库

3. **应用安全**：
   - 定期更新依赖包
   - 启用输入验证
   - 防止 SQL 注入
   - 防止 XSS 攻击
